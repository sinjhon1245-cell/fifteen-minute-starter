# 15분만

> 하나만 고르고, 15분만.

**15분만**은 자기계발 목표를 세우고 계획하는 앱이 아닙니다. 앱이 미리 정해 둔 작은 행동 하나를 오늘 15분 동안 실행하게 만드는 모바일 전용 PWA입니다. 끝나면 멈추고, 다음에 이어갑니다.

## 핵심 철학

- 계획보다 실행
- 의욕보다 시작
- 완벽보다 완료
- 몰아서 하는 것보다 반복
- 하루 한 단계, 15분이 끝나면 멈춤
- 많이 한 시간이 아니라 다시 시작한 날을 기록

## 주요 기능

- 8개 자기계발 분야, 16개 목표, 80개의 현실적인 15분 단위 실행 단계
- expectedEndTime 기반의 정확한 15분 고정 타이머 (새로고침·백그라운드 전환 후에도 정확히 복구)
- 페이지 숨김(다른 앱 이동) 감지 및 제한된 이탈 알림
- "먼저 끝냈어요" — 정한 행동을 일찍 마쳤을 때 실행 시간을 기록하고 바로 결과를 남기는 흐름
- "이번 실행을 멈출까요?" — 계속하기 / 행동 한 번 줄이기(세션당 1회 제한) / 지금까지 기록하고 종료하기 3가지 선택 카드
- 실행 결과를 4단계(완료 / 필요한 만큼 진행 / 일부만 진행 / 시작만 함)로 기록하는 결과 기록 화면
- 실제 실행 시간(actualDurationMs) 기반 기록 — 15분을 다 채우지 못해도 실행한 시간이 남습니다
- 같은 목표는 하루 한 번만 실행(3초 길게 눌러야 예외적으로 재실행 가능)
- 홈 화면에 즉시 설치할 수 있는 설치 카드 (beforeinstallprompt 지원 시 한 번의 탭으로 설치)
- Service Worker 기반 오프라인 캐싱, GitHub Pages 하위 경로 대응
- 서버, 로그인, 광고, 결제, 외부 API 없이 전부 로컬 저장소(localStorage)로 동작

## 사용 흐름

1. 자기계발 분야 선택
2. 구체적인 목표 선택
3. 오늘의 작은 행동 확인
4. "15분 시작" 클릭 → 즉시 타이머 시작
5. 한 행동에 집중 (필요하면 먼저 끝내거나, 행동을 한 번 줄이거나, 중간에 기록하고 종료 가능)
6. 15분 종료 알림
7. 하던 일을 정리
8. 이번 실행에서 어디까지 진행했는지 기록
9. 다음 실행 때 다음 단계 진행

## 자기계발 분야

독서, 운동, 공부, 영어, 글쓰기, 공간 및 디지털 정리, 생활 관리, 재정 관리 — 각 분야마다 핵심 목표 2개, 각 목표마다 5단계의 실제적인 15분 행동이 준비되어 있습니다. 페이지 수·문제 수 등을 강요하지 않고, 15분 안에 실제로 끝낼 수 있는 행동과 눈에 보이는 결과 중심으로 구성했습니다.

## 로컬 실행 방법

Node.js나 별도 패키지 설치 없이 실행할 수 있습니다.

### 방법 1. PowerShell 정적 서버 (권장, Windows)

```powershell
powershell -ExecutionPolicy Bypass -File scripts/serve.ps1 -Port 5173
```

브라우저에서 `http://localhost:5173` 접속.

### 방법 2. 다른 정적 서버

Python이나 Node.js가 설치되어 있다면 원하는 정적 서버로 프로젝트 루트를 서빙해도 됩니다. Service Worker와 알림·Wake Lock 관련 API는 `https://` 또는 `http://localhost` 같은 보안 컨텍스트에서만 정상 동작하므로 `file://`로 직접 여는 방식은 권장하지 않습니다.

## 데이터 검증

목표 데이터의 무결성(분야 8개, 목표 16개 이상, 목표당 단계 4개 이상, 전체 단계 70개 이상, 빈 필드·중복 ID 없음)은 `scripts/validate.html`을 브라우저로 열면 자동으로 검사되어 결과가 표시됩니다.

## GitHub Pages 배포 방법

1. 이 저장소를 GitHub에 공개 저장소로 push합니다.
2. GitHub 저장소의 Settings → Pages에서 배포 브랜치를 `main`, 폴더를 `/ (root)`로 설정합니다.
3. `https://<계정>.github.io/<저장소명>/` 주소로 접속해 정상 동작을 확인합니다.
4. 모든 리소스 경로(`manifest.webmanifest`, `service-worker.js`, CSS/JS/데이터/아이콘)는 상대 경로(`./`)로 작성되어 있어 저장소 이름이 무엇이든 하위 경로에서 그대로 동작합니다.

## 홈 화면 설치 방법

- 홈 화면 상단의 "홈 화면에 설치하기" 카드나 설정의 같은 메뉴를 누르면, 설치 가능한 환경(Android Chrome 등)에서는 별도 화면 없이 바로 시스템 설치창이 열립니다.
- 이미 설치되어 실행 중이면 설치 카드는 자동으로 숨겨집니다.
- 설치 이벤트를 지원하지 않는 브라우저에서는 버튼을 누르면 해당 브라우저에 맞는 짧은 수동 설치 안내가 표시됩니다. (예: "Chrome 메뉴에서 '앱 설치' 또는 '홈 화면에 추가'를 선택하세요.")

## 알림 권한 안내

- 알림 권한은 앱 진입 시 바로 요청하지 않습니다. 사용자가 처음으로 "15분 시작"을 누른 직후, 타이머 실행을 막지 않는 바텀시트로 안내합니다.
- "종료 알림 받기"를 선택하면 시스템 권한 요청 결과(허용/거부/닫기)와 관계없이 바텀시트가 즉시 닫히고, 짧은 토스트로 결과만 안내합니다. 사용자가 창을 다시 닫을 필요가 없습니다.
- "앱 화면으로만 알림 받기"를 선택해도 바텀시트가 즉시 닫힙니다.
- 한 번 선택한 사용자에게는 반복해서 묻지 않으며, 설정 화면의 "변경" 버튼으로 언제든 다시 바꿀 수 있습니다.
- 알림을 거부하거나 브라우저가 알림을 지원하지 않아도 타이머, 종료 화면, 종료음, 진동(지원 시), 실행 기록은 정상적으로 동작합니다.

## 웹 알림의 기술적 제한

이 앱은 서버 없이 브라우저 기능만으로 동작하는 PWA입니다. 다음 사항을 명확히 인지하고 사용해야 합니다.

- 브라우저나 PWA가 완전히 종료된 상태에서는 정확한 알림을 보장할 수 없습니다.
- 운영체제가 절전을 위해 백그라운드 웹앱 프로세스를 종료하면 종료 알림이 늦어지거나 표시되지 않을 수 있습니다.
- 휴대전화를 재부팅하면 진행 중이던 타이머는 복구되지 않습니다.
- Android 기본 알람 앱과 같은 전체 화면 알람, 시스템 알람 권한, 다른 앱 위 오버레이 기능은 제공하지 않습니다.
- 다른 앱의 이름이나 사용 시간을 감지하지 않으며, 특정 앱을 차단하지도 않습니다.
- 가장 확실한 사용을 위해서는 15분 동안 타이머 화면을 유지하는 것을 권장합니다.
- 종료 알림을 클릭하면 실행 중이던 PWA 창이 있으면 그 창을 앞으로 가져오고, 없으면 새로 열어 종료 또는 결과 기록 화면으로 이동시킵니다. 다만 브라우저·운영체제 정책에 따라 클릭 후 동작이 달라질 수 있습니다.

## 실제 실행 시간과 결과 기록

- 모든 세션은 `startTime`과 `actualEndTime`의 차이로 실제 실행 시간(`actualDurationMs`)을 계산해 저장합니다. 15분을 다 채운 경우에만 "15분"으로 표시되고, 그 외에는 실제로 실행한 시간(예: "14분 5초", "42초")이 표시됩니다.
- 타이머가 자연 종료되거나, "먼저 끝냈어요"로 일찍 마치거나, "이번 실행을 멈출까요?"에서 "지금까지 기록하고 종료하기"를 선택하면 모두 같은 결과 기록 화면으로 이동합니다.
- 결과 기록 화면에서 "정한 행동을 마쳤어요 / 필요한 만큼 진행했어요 / 일부만 진행했어요 / 시작만 했어요" 중 하나를 선택하고 "기록하고 마무리하기"를 누르면 한 번의 탭으로 기록이 끝납니다. 추가 메모 입력은 없습니다.
- "행동 줄이기"는 한 세션에서 한 번만 사용할 수 있습니다. 이미 사용한 세션에서는 버튼이 잠금 아이콘과 함께 비활성화되며, 새로고침 후에도 그 상태가 유지됩니다.

## 화면 유지 기능 (Wake Lock)

타이머 실행 중 지원되는 브라우저에서 Screen Wake Lock API를 사용해 화면이 자동으로 꺼지는 것을 방지합니다. 미지원 기기에서도 앱은 정상 작동하며, 설정에서 이 기능을 끌 수 있습니다.

## 데이터 저장 위치와 개인정보 처리

- 회원가입과 로그인이 없습니다.
- 모든 실행 기록과 설정은 현재 브라우저(또는 설치된 PWA)의 `localStorage`에만 저장됩니다.
- 어떤 개인정보나 사용 기록도 외부 서버로 전송되지 않습니다. 이 앱은 서버 자체가 없습니다.
- 브라우저 데이터(사이트 데이터)를 삭제하면 기록도 함께 삭제됩니다.
- 다른 기기와 자동으로 동기화되지 않습니다.

## 목표 데이터 수정 방법

모든 목표와 단계 데이터는 `data/goals.js` 한 파일에 UI 코드와 분리되어 있습니다. `GOALS.push(goal(...))` 형태로 목표를 추가하고, 각 목표의 `steps` 배열에 `step(...)` 형태로 단계를 추가합니다. 각 단계는 `action`, `preparation`, `focusMessage`, `fallbackAction`, `finishAction`, `nextPreview` 여섯 개 필드를 모두 채워야 합니다.

## 새로운 목표 및 단계 추가 방법

1. `data/goals.js`에서 `CATEGORIES` 배열에 새 분야를 추가하거나 기존 분야를 사용합니다.
2. 새 목표는 `GOALS.push(goal(id, categoryId, title, description, startMessage, steps))` 형태로 추가합니다.
3. 각 목표는 4~6단계를 갖도록 `step(...)`을 채웁니다. 페이지 수·문제 수 등을 강제하지 말고, 15분 안에 실제로 끝낼 수 있는 하나의 행동만 담습니다.
4. `scripts/validate.html`을 브라우저로 열어 데이터 무결성(빈 필드, 중복 ID, 최소 개수 조건)을 확인합니다.

## 기존 목표가 삭제·통합된 경우

목표 ID가 바뀌거나 사라지면, 해당 ID를 참조하던 예전 실행 기록은 기록 화면에 계속 표시됩니다(기록 저장 시점의 목표명·단계명을 함께 저장해 두기 때문입니다). 다만 그 목표로 다시 들어가는 링크는 더 이상 존재하지 않으므로, 홈이나 분야 화면에서 현재 제공되는 목표를 다시 선택하면 됩니다. 존재하지 않는 목표를 직접 열려고 하면 안내 토스트와 함께 홈으로 이동합니다.

## 테스트 방법

Node.js 없이도 검증할 수 있도록 구성했습니다.

- **데이터 무결성 검증**: `scripts/validate.html`을 브라우저로 열면 분야 수, 목표 수, 전체 단계 수, 빈 필드, 중복 ID를 자동 검사합니다.
- **기능 테스트**: `scripts/serve.ps1`로 로컬 서버를 띄운 뒤 브라우저에서 온보딩 → 목표 선택 → 15분 시작 → 타이머 → 종료 → 결과 기록까지 실제로 눌러보며 확인합니다.
- **개발용 타이머 단축**: `localhost`나 `file://`에서만 `?dev_timer_seconds=10`과 같은 쿼리 파라미터로 타이머 길이를 초 단위로 줄여 테스트할 수 있습니다. 이 값은 실제 배포 도메인(GitHub Pages)에서는 무시되며 항상 15분으로 고정됩니다.

## 알려진 제한 사항

- 이 앱은 브라우저/PWA가 완전히 종료된 상태에서의 정확한 알림을 보장하지 않습니다.
- 다른 앱 사용을 감지하거나 차단하는 기능은 없습니다.
- 여러 기기 간 기록 동기화 기능은 없습니다(로컬 저장소 기반).
- 15분이라는 실행 시간은 고정되어 있으며 사용자가 변경할 수 없습니다.

## 향후 Android 앱으로 전환할 경우 추가 가능한 기능

- 시스템 알람 권한을 사용한 완전 종료 상태에서의 정확한 알림
- 다른 앱 위 오버레이를 이용한 강제 집중 모드
- 특정 앱 실행 감지 및 일시 차단
- 여러 기기 간 클라우드 동기화
- 홈 화면 위젯을 통한 오늘의 행동 미리보기
