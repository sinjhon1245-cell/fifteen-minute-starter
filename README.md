# 15분 시동

> 변명 끝. 15분 시작.

**15분 시동**은 자기계발 목표를 세우고 계획하는 앱이 아닙니다. 앱이 미리 정해 둔 작은 행동 하나를 오늘 15분 동안 실행하게 만드는 모바일 전용 PWA입니다.

## 핵심 철학

- 계획보다 실행
- 의욕보다 시작
- 완벽보다 완료
- 몰아서 하는 것보다 반복
- 하루 한 단계, 15분이 끝나면 멈춤
- 많이 한 시간이 아니라 다시 시작한 날을 기록

## 주요 기능

- 8개 자기계발 분야, 24개 이상의 구체적인 목표, 168개 이상의 15분 단위 실행 단계
- expectedEndTime 기반의 정확한 15분 고정 타이머 (새로고침·백그라운드 전환 후에도 정확히 복구)
- 페이지 숨김(다른 앱 이동) 감지 및 제한된 이탈 알림
- 중도 종료 확인 → 행동 절반 축소 → 포기 확인으로 이어지는 3단계 종료 퍼널
- 같은 목표는 하루 한 번만 실행(3초 길게 눌러야 예외적으로 재실행 가능)
- Service Worker 기반 오프라인 캐싱, GitHub Pages 하위 경로 대응
- 서버, 로그인, 광고, 결제, 외부 API 없이 전부 로컬 저장소(localStorage)로 동작

## 사용 흐름

1. 자기계발 분야 선택
2. 구체적인 목표 선택
3. 오늘의 작은 행동 확인
4. "15분 시작" 클릭 → 즉시 타이머 시작
5. 한 행동에 집중
6. 15분 종료 알림
7. 하던 일을 정리
8. 오늘 단계 종료
9. 다음 실행 때 다음 단계 진행

## 자기계발 분야

독서, 운동, 공부, 영어, 글쓰기, 공간 및 디지털 정리, 생활 관리, 재정 관리 — 각 분야마다 3개 이상의 목표, 각 목표마다 7단계의 15분 행동이 준비되어 있습니다.

## 기술 구조

- 순수 HTML / CSS / JavaScript, 빌드 과정 없음
- 프레임워크(React, Vue 등) 미사용
- 해시 기반 SPA 라우터
- 데이터(목표·단계)와 화면 렌더링·저장·타이머·알림·Wake Lock·PWA 로직을 파일 단위로 분리

## 로컬 실행 방법

Node.js나 별도 패키지 설치 없이 실행할 수 있습니다.

### 방법 1. PowerShell 정적 서버 (권장, Windows)

```powershell
powershell -ExecutionPolicy Bypass -File scripts/serve.ps1 -Port 5173
```

브라우저에서 `http://localhost:5173` 접속.

### 방법 2. 다른 정적 서버

Python이나 Node.js가 설치되어 있다면 원하는 정적 서버로 프로젝트 루트를 서빙해도 됩니다. Service Worker와 알림·Wake Lock 관련 API는 `https://` 또는 `http://localhost` 같은 보안 컨텍스트에서만 정상 동작하므로 `file://`로 직접 여는 방식은 권장하지 않습니다.

## 데이터 검증

목표 데이터의 무결성(분야 8개, 목표 24개 이상, 목표당 단계 7개 이상, 전체 단계 168개 이상, 빈 필드·중복 ID 없음)은 `scripts/validate.html`을 브라우저로 열면 자동으로 검사되어 결과가 표시됩니다.

## GitHub Pages 배포 방법

1. 이 저장소를 GitHub에 공개 저장소로 push합니다.
2. GitHub 저장소의 Settings → Pages에서 배포 브랜치를 `main`, 폴더를 `/ (root)`로 설정합니다.
3. `https://<계정>.github.io/<저장소명>/` 주소로 접속해 정상 동작을 확인합니다.
4. 모든 리소스 경로(`manifest.webmanifest`, `service-worker.js`, CSS/JS/데이터/아이콘)는 상대 경로(`./`)로 작성되어 있어 저장소 이름이 무엇이든 하위 경로에서 그대로 동작합니다.

## PWA 설치 방법 (Android Chrome)

1. Chrome 메뉴 열기
2. "앱 설치" 또는 "홈 화면에 추가" 선택
3. 설치 후 홈 화면 아이콘으로 실행

`beforeinstallprompt`를 지원하는 환경에서는 앱 내부 설정 화면에서 바로 설치 버튼을 제공합니다.

## 알림 권한 안내

- 알림 권한은 앱 진입 시 바로 요청하지 않습니다. 사용자가 처음으로 "15분 시작"을 누른 직후, 타이머 실행을 막지 않는 바텀시트로 안내합니다.
- "알림 허용"을 선택한 경우에만 브라우저 알림 권한을 요청합니다.
- 알림을 거부하거나 브라우저가 알림을 지원하지 않아도 타이머, 종료 화면, 종료음, 진동(지원 시), 실행 기록은 정상적으로 동작합니다.

## 웹 알림의 기술적 제한

이 앱은 서버 없이 브라우저 기능만으로 동작하는 PWA입니다. 다음 사항을 명확히 인지하고 사용해야 합니다.

- 브라우저나 PWA가 완전히 종료된 상태에서는 정확한 알림을 보장할 수 없습니다.
- 운영체제가 절전을 위해 백그라운드 웹앱 프로세스를 종료하면 종료 알림이 늦어지거나 표시되지 않을 수 있습니다.
- 휴대전화를 재부팅하면 진행 중이던 타이머는 복구되지 않습니다.
- Android 기본 알람 앱과 같은 전체 화면 알람, 시스템 알람 권한, 다른 앱 위 오버레이 기능은 제공하지 않습니다.
- 다른 앱의 이름이나 사용 시간을 감지하지 않으며, 특정 앱을 차단하지도 않습니다.
- 가장 확실한 사용을 위해서는 15분 동안 타이머 화면을 유지하는 것을 권장합니다.

## 화면 유지 기능 (Wake Lock)

타이머 실행 중 지원되는 브라우저에서 Screen Wake Lock API를 사용해 화면이 자동으로 꺼지는 것을 방지합니다. 미지원 기기에서도 앱은 정상 작동하며, 설정에서 이 기능을 끌 수 있습니다.

## 데이터 저장 위치와 개인정보 처리

- 회원가입과 로그인이 없습니다.
- 모든 실행 기록과 설정은 현재 브라우저(또는 설치된 PWA)의 `localStorage`에만 저장됩니다.
- 어떤 개인정보나 사용 기록도 외부 서버로 전송되지 않습니다. 이 앱은 서버 자체가 없습니다.
- 브라우저 데이터(사이트 데이터)를 삭제하면 기록도 함께 삭제됩니다.
- 다른 기기와 자동으로 동기화되지 않습니다.

## 목표 데이터 수정 방법

모든 목표와 단계 데이터는 `data/goals.js` 한 파일에 UI 코드와 분리되어 있습니다. `GOALS.push(goal(...))` 형태로 목표를 추가하고, 각 목표의 `steps` 배열에 `step(...)` 형태로 단계를 추가합니다. 각 단계는 `action`, `preparation`, `focusMessage`, `fallbackAction`, `finishAction`, `nextPreview` 여섯 개 필드를 모두 채워야 합니다.

## 새로운 목표 및 단계 추가 방법

1. `data/goals.js`에서 `CATEGORIES` 배열에 새 분야를 추가하거나 기존 분야를 사용합니다.
2. 새 목표는 `GOALS.push(goal(id, categoryId, title, description, startMessage, steps))` 형태로 추가합니다.
3. 각 목표는 최소 7단계, 최대 10단계를 갖도록 `step(...)`을 채웁니다.
4. `scripts/validate.html`을 브라우저로 열어 데이터 무결성(빈 필드, 중복 ID, 최소 개수 조건)을 확인합니다.

## 테스트 방법

Node.js 없이도 검증할 수 있도록 구성했습니다.

- **데이터 무결성 검증**: `scripts/validate.html`을 브라우저로 열면 분야 수, 목표 수, 전체 단계 수, 빈 필드, 중복 ID를 자동 검사합니다.
- **기능 테스트**: `scripts/serve.ps1`로 로컬 서버를 띄운 뒤 브라우저에서 온보딩 → 목표 선택 → 15분 시작 → 타이머 → 종료 → 기록까지 실제로 눌러보며 확인합니다.
- **개발용 타이머 단축**: `localhost`나 `file://`에서만 `?dev_timer_seconds=10`과 같은 쿼리 파라미터로 타이머 길이를 초 단위로 줄여 테스트할 수 있습니다. 이 값은 실제 배포 도메인(GitHub Pages)에서는 무시되며 항상 15분으로 고정됩니다.

## 알려진 제한 사항

- 이 앱은 브라우저/PWA가 완전히 종료된 상태에서의 정확한 알림을 보장하지 않습니다.
- 다른 앱 사용을 감지하거나 차단하는 기능은 없습니다.
- 여러 기기 간 기록 동기화 기능은 없습니다(로컬 저장소 기반).
